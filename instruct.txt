1. Introduction This module focuses on building a service listing and management system similar to platforms like Zomato, but for event-related services such as caterers, decorators, DJs, and venue providers. The goal is to provide users with a centralized platform where they can explore, compare, and select services based on their requirements, availability, and budget. 2. Objective To create a detailed service listing platform for event planning. To allow users to view, filter, and select services like catering, decoration, and DJ. To store all service-related data in a database for efficient retrieval. To provide user authentication (login system) for secure access. To implement budget-based filtering for better decision-making. 3. System Features 3.1 Service Categories The system will include multiple categories: Caterers Decorators DJ Services Venue Providers Each category will have a separate page displaying all available options. 3.2 Detailed Service Pages Each service provider will have: Name and description Available dates Services offered (menu, decoration types, music, etc.) Pricing details Category classification Availability status Example: A caterer will show: What food items they provide On which dates they are available Price per plate or package 3.3 Zomato-like UI Services will be displayed in card format with images Fake/demo images will be used for UI representation Clean and user-friendly interface 3.4 Database Management All data (services, users, bookings) will be stored in MongoDB Proper schema design for: Users Services Categories Availability Ensures no data errors and smooth functioning 3.5 Login & Authentication Users can register and login using email ID Secure authentication system Predefined login for testing (dummy credentials) Real email login support (optional integration) 3.6 Filtering System Users can filter services based on: Budget Category Availability dates Backend filtering logic implemented using Node.js 3.7 Availability Management Each service will show: Available dates Booked dates Helps users make quick decisions 4. Technology Stack Frontend: React.js Backend: Node.js / Express.js Database: MongoDB API: REST APIs 5. Working Flow User logs into the system Selects a category (Caterer / DJ / Decorator) Views detailed service listings Applies filters (budget, date, etc.) Checks availability and details Selects preferred service 6. Advantages Saves time in event planning All services available in one platform Easy comparison of providers Budget-friendly filtering Organized and structured data 7. Conclusion This system simplifies event planning by providing a centralized, user-friendly, and data-driven platform. It enhances decision-making by offering detailed insights, availability tracking, and budget-based filtering, making it more efficient than traditional method Additional Feature: User Authentication (Login System) Users must register/login using email ID and password Secure authentication using backend (Node.js) User-specific data will be stored in the database Ensures personalized experience and data security 2. Wishlist Feature Users can save services (Caterers, DJs, Decorators, Venues) to a wishlist Helps users compare and decide later Wishlist data is stored in MongoDB linked to user account Accessible anytime after login 3. Add to Cart Feature Users can add selected services to cart before finalizing Cart includes: Selected service Price details Selected date Helps in planning and managing multiple services together
